# Top 10 Global Banks — Financial Health Dashboard
### A beginner data science project using Excel, Java, JavaScript, and Power BI

## The research question

> **How does the overall financial health — profitability, capital strength,
> efficiency, and momentum — of the largest European banks compare with the
> largest American banks, and how has it evolved from 2021 to 2025?**

**Banks covered (Top 5 Europe + Top 5 America):**

| Region | Banks |
|---|---|
| Americas | JPMorgan Chase, Bank of America, Citigroup, Wells Fargo, Goldman Sachs |
| Europe | BNP Paribas, Banco Santander, Deutsche Bank, Barclays, UBS Group |

## How to run the pipeline yourself

You only need to do this if you want to **change the data** (add a bank,
update a year, fix a number). If you just want to build the dashboard,
skip to "Build the Power BI dashboard" below — the data files are already
generated for you.

### 1. Edit the raw data (optional)
Edit `data/financials.csv`, `data/banks.csv`, or `data/fx_rates.csv` in
Excel or any text editor. Keep the column headers identical.

### 2. Run the Java program
Requires a Java JDK (version 11+). Check with `java -version` / `javac -version`.

```bash
cd java
javac BankFinancialAnalyzer.java
java BankFinancialAnalyzer
```

This regenerates `data/processed_bank_data.csv` and prints a ranked list of
banks by Financial Health Score to the console.

**What the Financial Health Score means:** a 0–100 score per bank, per year,
combining (within that year, across all 10 banks):
- 35% Return on Equity (profitability)
- 25% CET1 ratio (capital safety buffer)
- 20% Net profit margin (efficiency)
- 20% Net income year-over-year growth (momentum)

It's a relative ranking tool, not an absolute "grade" — a bank scores
high if it's doing well **compared to its peers that year**.

### 3. Run the Node.js validation script
Requires Node.js (version 16+). Check with `node --version`.

```bash
cd js
node validate_and_summarize.js
```

This checks for missing values, duplicate rows, and **statistical outliers**
(z-score > 2). For example, it correctly flags **UBS's 2023 Net Income** as
an outlier — that year included a one-time multi-billion-dollar accounting
gain ("negative goodwill") from acquiring Credit Suisse, which is a real
and useful thing to call out on your dashboard rather than something to
hide. This is a good talking point in your project write-up: real financial
data has one-off events that distort year-over-year comparisons, and a good
analyst flags them rather than ignoring them.

### 4. Open the preview dashboard (optional sanity check)
Open `js/preview_dashboard.html` directly in a browser, or serve it locally
to avoid browser file-access restrictions:

```bash
cd js
npx serve .
```

Then visit the printed localhost URL. This is **not** your final deliverable
— it's a 2-minute visual check that the numbers look reasonable before you
invest time building the polished Power BI report.

### 5. Rebuild the Excel workbook (only if you changed the CSVs)
Requires Python 3 with `openpyxl` installed (`pip install openpyxl`).
```bash
cd ..
python3 build_excel.py
```

### 6. Build the Power BI dashboard
Follow **docs/PowerBI_Guide.md** step by step. It walks through importing
`bank_financial_data.xlsx`, writing the DAX measures, and building the map,
KPI cards, and trend charts.

## Data sources & important caveats

- Figures for **2024 and 2025 were verified** against each bank's official
  full-year earnings press releases and annual reports (JPMorgan, Bank of
  America, Citigroup, Wells Fargo, Goldman Sachs, BNP Paribas, Santander,
  Deutsche Bank, Barclays, UBS — all published Jan–Mar 2026 for FY2025).
- Figures for **2021–2023** are compiled from each bank's published annual
  results and are accurate to the best of available knowledge, but were not
  individually re-verified line-by-line. **If this project will be formally
  graded or cited, cross-check the exact numbers against each bank's 10-K /
  20-F filing or annual report before submitting** — banks occasionally
  restate prior-year figures (e.g., BNP Paribas restated its 2024 series in
  March 2026), so "officially reported at the time" and "restated today"
  can differ slightly.
- **FX rates** (EUR→USD, GBP→USD) used for currency normalization are
  approximate annual averages, not exact daily/period rates. For a more
  rigorous treatment, source exact rates from the ECB or Federal Reserve
  historical data series.
- **UBS reports natively in USD** (no conversion needed). **BNP Paribas and
  Deutsche Bank report in EUR. Santander reports in EUR. Barclays reports
  in GBP.** All are converted to USD in `Financials_USD` and
  `Processed_for_PowerBI` for fair comparison.
- Total Assets, Employees, and 2025 figures for some banks are reasonable
  approximations based on year-end disclosures; treat any number you plan
  to quote precisely as something to verify against the primary source.

## Suggested write-up structure (for your project report)

1. **Introduction** — the research question and why comparing EU vs US banks
   matters (different regulatory regimes — Basel III/CRR in the EU/UK/CH vs
   US capital rules — different interest rate cycles, different M&A activity
   like UBS–Credit Suisse).
2. **Methodology** — describe the 4-tool pipeline (this README's diagram).
3. **Data** — describe sources, the Financial Health Score formula, and the
   currency-normalization caveat.
4. **Findings** — screenshots of your Power BI dashboard, with 3-4 key
   observations (e.g., "US banks generally posted higher ROE but Europe's
   capital ratios converged with the US by 2025", "UBS's 2023 figures are an
   outlier due to the Credit Suisse acquisition").
5. **Limitations** — approximate FX rates, non-audited early-year figures,
   only 10 of dozens of major global banks.
6. **Conclusion**.
